## 4. 统一错误码表

### 4.1 系统级错误码

| 错误码 | HTTP 状态码 | 说明 | 处理建议 |
|--------|------------|------|---------|
| 0 | 200 | 成功 | - |
| INTERNAL_ERROR | 500 | 服务内部错误 | 稍后重试 |
| PARAM_INVALID | 400 | 参数校验失败 | 检查请求参数 |
| UNAUTHORIZED | 401 | 未认证 | 重新登录 |
| FORBIDDEN | 403 | 无权限 | 联系管理员 |
| NOT_FOUND | 404 | 资源不存在 | 检查资源 ID |
| METHOD_NOT_ALLOWED | 405 | 请求方法不允许 | 检查 HTTP 方法 |
| TOO_MANY_REQUESTS | 429 | 请求过于频繁 | 降低请求频率 |

| 错误码 | HTTP 状态码 | 说明 | 处理建议 |

| USER_ALREADY_EXISTS | 409 | 用户已存在 | 更换用户名/邮箱 |
| INVALID_CREDENTIALS | 401 | 用户名或密码错误 | 检查登录信息 |
| TOKEN_EXPIRED | 401 | 令牌已过期 | 刷新令牌或重新登录 |

## 5. 接口详情

### 5.1 POST /api/v1/auth/login

<!-- 用户登录接口 → {{功能缩写}}-AC-{{序号}} -->

**描述**：用户通过用户名和密码登录系统，获取访问令牌。

**请求参数**：

| 字段名 | 类型 | 位置 | 必填 | 说明 | 校验规则 |
|--------|------|------|------|------|---------|
| username | string | Body | 是 | 用户名 | 3-50 个字符 |
| password | string | Body | 是 | 密码 | 6-20 个字符 |

**请求示例**：

```json
{
    "username": "example_user",
    "password": "example_pass"
}
```

**成功响应**（200 OK）：

```json
{
    "code": 0,
    "message": "success",
    "data": {
        "access_token": "eyJhbGciOiJIUzI1NiIs...",
        "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
        "expires_in": 7200,
        "user": {
            "id": 1,
            "username": "example_user",
            "nickname": "示例用户"
        }
    }
}
```

**错误场景**：

| HTTP 状态码 | 错误码 | 说明 | 触发条件 |
|------------|--------|------|---------|
| 400 | PARAM_INVALID | 参数校验失败 | 用户名或密码为空 |
| 401 | INVALID_CREDENTIALS | 认证失败 | 用户名或密码错误 |
| 404 | USER_NOT_FOUND | 用户不存在 | 用户名未注册 |

## 7. 限流规则

<!-- 仅当需要接口级别的限流时填写本章节。如果不需要，填写"本项目暂无限流需求"即可。 -->

### 7.1 限流策略

| 限流维度 | 限流算法 | 限流阈值 | 超限响应 |
|---------|---------|---------|---------|
| 全局 | {{如：令牌桶}} | {{如：1000 次/分钟}} | 返回 429 Too Many Requests |
| 单用户 | {{如：滑动窗口}} | {{如：100 次/分钟}} | 返回 429 + Retry-After Header |
| 单 IP | {{如：固定窗口}} | {{如：200 次/分钟}} | 返回 429 + Retry-After Header |

### 7.2 接口级别限流

| 接口路径 | 限流阈值 | 说明 |
|---------|---------|------|
| POST /api/v1/auth/login | {{如：10 次/分钟/IP}} | 防止暴力破解 |
| POST /api/v1/auth/register | {{如：5 次/分钟/IP}} | 防止恶意注册 |
| POST /api/v1/{{resource}} | {{如：30 次/分钟/用户}} | 防止数据滥用 |

### 7.3 超限响应

```json
{
    "code": "TOO_MANY_REQUESTS",
    "message": "请求过于频繁，请稍后再试",
    "details": {
        "retry_after": 60
    }
}
```

**响应 Header**：

| Header | 说明 | 示例 |
|--------|------|------|
| Retry-After | 建议重试等待时间（秒） | 60 |
| X-RateLimit-Limit | 限流阈值 | 100 |
| X-RateLimit-Remaining | 剩余可用次数 | 95 |
| X-RateLimit-Reset | 限流窗口重置时间（Unix 时间戳） | 1713523200 |
