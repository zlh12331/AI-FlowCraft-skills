# 更新日志

本文件记录了 AI-FlowCraft 的所有重要变更。

## [V1.1.0] - 2026-05-12

### 新增

#### 新目录
- 测试相关 Skill（22-26）新增 `examples/` 目录，包含测试示例文件：
  - `test-examples.md`（Skill 22）
  - `component-test-examples.md`（Skill 23）
  - `integration-test-examples.md`（Skill 24）
  - `e2e-test-examples.md`（Skill 25）
  - `system-test-examples.md`（Skill 26）
- Skill 26-test-system 的 `resources/` 目录新增 `CHANGELOG.md`
- 每个 Skill 目录下新增对应的 `.zip` 压缩包（共 28 个）

#### SKILL.md 新增章节
- **使用场景**章节：定义每个 Skill 的适用场景
- **不适用场景**子章节：引导用户使用正确的 Skill
- **指令**章节：明确所需的输入和输出
- 测试相关 Skill 新增**交互准则**章节

#### GUARDRAILS.md 新增规则
- **规则 9**：实现与设计文档一致性
  - 编码阶段产出的代码必须与设计阶段的 specs 文档保持一致
  - 涵盖 API 路径、响应格式、数据模型字段、业务规则等
  - 任何不一致必须记录为偏差，不得静默偏离
- **第 7 节**：全局失败处理
  - 前置条件不满足时的处理
  - 执行过程中出错时的处理
  - 输出不符合预期时的处理

#### 测试 Skill（22-26）新增功能
- **规则 12**：设计文档驱动的业务规则覆盖
  - 读取后端技术方案中的"Service 层业务规则"章节
  - 为每条业务规则编写至少一个正向测试和一个异常/边界测试
  - 在测试报告中新增"业务规则覆盖"章节

### 变更

#### 目录结构
- `assets/` → `templates/`（模板文件目录）
- `scripts/` → `resources/`（测试脚本目录）

#### SKILL.md 格式
- 简化 YAML frontmatter（移除 `allowed-tools`、`metadata` 字段）
- 重新组织章节层级：
  - "项目上下文协议"和"边界守卫"变为"描述"章节的子章节
  - "工作流程"变为"指令"章节的子章节
- 移除 Skill 1 中独立的"文档质量保证"章节

#### README.md
- 更新项目结构图，增加 V1 版本目录层级
- 更新文件统计表格格式

### 修复
- 提升 Skill 文档结构的一致性
- 更好地引导用户在正确场景使用正确的 Skill

## [V1.0.1] - 2026-05-03

### 新增
- AI-FlowCraft 初始版本发布，包含 28 个专业 Skill
- 完整的工作流系统，从需求到测试全覆盖
- 文档驱动开发方法
- AC（验收标准）全链路追溯
- 边界守卫机制
- TDD 驱动开发流程
- 五层测试金字塔
- 多技术栈适配

### 包含的 Skills
- **项目启动（Skill 1-18）**：需求澄清、PRD 生成、系统架构、数据库设计、API 设计等
- **功能开发（Skill 19-21）**：任务规划、编码实现、阶段报告
- **测试（Skill 22-26）**：单元测试、组件测试、集成测试、E2E 测试、系统测试
- **随时可用（Skill 27-28）**：功能变更、Bug 修复

### 文档
- 28 个 SKILL.md 文件
- 24 个输出模板
- 25 个测试脚本
- 2 个全局协议文件（GUARDRAILS.md、PROJECT-CONTEXT.md）
- 中英文 README 文件
